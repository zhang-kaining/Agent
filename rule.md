## 代码开发规范

### 代码质量标准
• 严格遵循语言编码规范和最佳实践
• 坚持最小改动原则，仅修改必要部分
• 直接解决核心问题，不回避或绕过
• 每个函数控制在40行以内
• 代码保持单行完整，避免不必要换行
• 函数参数采用多行格式，每个参数占一行
    eg. def foo(  # 括号单独一行
        bar,
        baz,
        qux
    ):  # 括号单独一行
        pass

### 文件组织管理
• 项目根目录禁止创建临时文件或测试文件
• 临时文件统一放置在 temp/ 目录
• 测试文件按规定目录结构组织
• 脚本文件统一存放在指定文件夹，不按版本分散
• 及时清理错误文件，保持代码库整洁

### 代码编写原则
• 每次代码写入量控制到最少
• 实现方式追求简单高效
• 复杂功能拆分为小模块
• 脚本长度合理控制
• 优先选择直接有效的解决方案

### 文档策略
• 不主动创建 Markdown 文档
• 减少冗余文档生成
• 优先在代码中添加注释
• 文档内容保持简洁实用
• 讨论方案时不输出代码示例

### 工作流程
• 方案讨论阶段专注于逻辑分析
• 实现阶段严格按规范编码
• 代码审查重点关注最小化原则
• 持续重构保持代码质量

### 开发环境配置
• 使用 uv 管理 Python 虚拟环境
• 项目根目录必须包含 venv 虚拟环境
• 依赖管理统一通过 uv 工具处理
